# AIOS CLI User Guide & Tutorial

This guide teaches you how to use the newly packaged **AIOS CLI (`aios`)** to run multi-agent workflows across different workspace directories on your computer.

---

## 1. How It Works (Editable Mode)

The package is installed in **Editable Mode (`-e`)**. This means:
* You can edit agent profiles (e.g. in `agents/`) or python files (in `runtime/`) directly inside the `AI Agentic` folder.
* Any changes you make will **instantly take effect** the next time you run the `aios` command anywhere on your computer. No need to reinstall the package!

---

## 2. Installation Steps (Recap)

We have set up a virtual environment inside your `AI Agentic` folder to prevent permissions issues.

1. **Activate the Virtual Environment** (in your terminal inside the `AI Agentic` directory):
   ```bash
   source .venv/bin/activate
   ```
2. **(Optional) Re-installing/Updating**:
   If you ever need to reinstall:
   ```bash
   pip install -e .
   ```

---

## 3. How to Use the `aios` Command

You can run workflows by calling `aios` (or `.venv/bin/aios` if the virtual environment is not activated).

### CLI Parameters

| Flag | Parameter | Description |
| :--- | :--- | :--- |
| `-w` | `--workflow` | **[Required]** Path to the workflow YAML file (checks local workspace first, then falls back to default workflows in the package). |
| `-s` | `--workspace` | **[Optional]** Target project directory where artifacts are read and written (default: current directory `.`). |
| `--agents-dir` | `--agents-dir` | **[Optional]** Path to agent profiles directory (defaults to packaged `agents/` folder). |
| `--memory-dir` | `--memory-dir` | **[Optional]** Path to shared memory directory (defaults to `<workspace>/memory`). |

---

## 4. Practical Examples

### Example A: Running a Bug Fix Pipeline on a Project
Suppose you want to debug an application located at `/Users/phutawanmueangma/Documents/Project/TarotCardReading`:

```bash
aios -w bug_fix_pipeline.yaml -s /Users/phutawanmueangma/Documents/Project/TarotCardReading
```

**What the CLI does:**
1. Loads [bug_fix_pipeline.yaml](file:///Users/phutawanmueangma/Documents/Project/AI%20Agentic%20/workflows/bug_fix_pipeline.yaml) from package templates.
2. Creates `/Users/phutawanmueangma/Documents/Project/TarotCardReading/memory` if not already present.
3. Automatically routes each stage (e.g., `qa_engineer` under `validation`, `fullstack_engineer` under `engineering`).
4. Loads agent profiles and runs the stages.
5. Generates the output files (`triaged_bug`, `bug_fix_patch`, `test_verdict`) inside the target directory `/Users/phutawanmueangma/Documents/Project/TarotCardReading`.
6. Appends a chronological decision log in `/Users/phutawanmueangma/Documents/Project/TarotCardReading/memory/technical_decisions.md`.

---

### Example B: Starting a Brand New Project Workspace
If you have a new project called `my-new-app`, you don't need to copy any AIOS folders. Just create an empty folder and run the CLI pointing to it:

1. Create a new directory:
   ```bash
   mkdir -p "/Users/phutawanmueangma/Documents/Project/my-new-app"
   ```
2. Run the website generation pipeline:
   ```bash
   aios -w website_generation.yaml -s "/Users/phutawanmueangma/Documents/Project/my-new-app"
   ```
3. Check the folder `/Users/phutawanmueangma/Documents/Project/my-new-app`—you will see `artifacts/`, `memory/`, and all output files generated by the agents!
